* **From:** `webctl:base@knot` (v0.2.0 small-refactoring step, 2026-06-22).

## What this does

Replaces five byte-duplicated `lib/` modules with thin re-export shims pointing
at the pinned `base-webctl` submodule (`vendor/base-webctl`). Each module is
**independently adoptable** — take them one at a time or as a batch.

| Module (consumer path) | base path | shim `require(...)` |
|---|---|---|
| `lib/browser-location/cdp-rewrite.js` | `lib/browser-location/cdp-rewrite.js` | `../../vendor/base-webctl/lib/browser-location/cdp-rewrite.js` |
| `lib/browser-location/localhost-direct.js` | `lib/browser-location/localhost-direct.js` | `../../vendor/base-webctl/lib/browser-location/localhost-direct.js` |
| `lib/xpra-presence.js` | `lib/xpra-presence.js` | `../vendor/base-webctl/lib/xpra-presence.js` |
| `lib/lru-cleanup.js` | `lib/lru-cleanup.js` | `../vendor/base-webctl/lib/lru-cleanup.js` |
| `lib/systemd-timer.js` | `lib/systemd-timer.js` | `../vendor/base-webctl/lib/systemd-timer.js` |

**Net diff across the five:** `+20 / -759` lines.

Proven headless in a throwaway clone of linkedin-webctl@`8e85d31`
(`/tmp/claude/260622-1545-webctl-base-pilot/linkedin-scratch`) with the submodule
pinned at base `9d9cfbd` (v0.2.0) and all five shims in place:

* **Full suite green:** all **24** `test/*-test.js` pass, **zero** failures —
  same as baseline. The modules consumed *through the submodule* include:
  * `lru-cleanup-test.js` → **112 passed**
  * `browser-location-test.js` → **23 passed** (exercises localhost-direct + cdp-rewrite)
  * `systemd-timer-test.js` → **10 passed**
  * `xpra-presence-test.js` → **5 passed**
  * `profile-lock-test.js` → **18 passed**, `docker-mode-test.js` → **57 passed**

These five are **pure / builtins-only with no internal deps and no top-level
await**, so the same synchronous `require(esm)` consumption path proven for
docker-ctl applies unchanged.

## The shim (identical pattern per module)

```js
// <name>.js — re-export shim (base-webctl submodule, sb7q v0.2.0).
// Implementation lives once in base; consumed via the pinned submodule.
// Node >=22.12 require(esm): base ships zero-dep ESM (no top-level await).
'use strict';
module.exports = require('<relative-path-from-table>');
```

Full patch: `/tmp/claude/260622-1545-webctl-base-pilot/linkedin-v0.2.0-five-modules.patch`.

## How to adopt (suggested)

```bash
cd linkedin-webctl
git checkout -b i-adopt-base-v0.2.0
# (submodule already added if you took the docker-ctl pilot; else:)
git submodule add git@github.com:CLIAI/base-webctl.git vendor/base-webctl
git -C vendor/base-webctl checkout <BASE_v0.2.0_SHA>
# replace each lib file above with its shim
node --test test/   # or: for t in test/*-test.js; do node "$t"; done  -> all green
git add .gitmodules vendor/base-webctl lib/...
git commit   # focused submodule-pin + shim commit (sb7q §version-bump)
```

## Preconditions on base (owner: webctl:base)
