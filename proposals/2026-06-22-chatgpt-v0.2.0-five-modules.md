## What this does

Same as the linkedin v0.2.0 proposal: replace five byte-duplicated `lib/`
modules with thin re-export shims pointing at `vendor/base-webctl`. Each is
independently adoptable.

| Module (consumer path) | base path | shim `require(...)` |
|---|---|---|
| `lib/browser-location/cdp-rewrite.js` | `lib/browser-location/cdp-rewrite.js` | `../../vendor/base-webctl/lib/browser-location/cdp-rewrite.js` |
| `lib/browser-location/localhost-direct.js` | `lib/browser-location/localhost-direct.js` | `../../vendor/base-webctl/lib/browser-location/localhost-direct.js` |

## Why it is safe — and the one honest caveat

These five files are **byte-identical between chatgpt-webctl and linkedin-webctl**
(verified 2026-06-22: `cmp` clean on all five). The base module was ported
verbatim from that shared source. Consumption was proven end-to-end against
**linkedin's** suite (24/24 green via the submodule — see the linkedin proposal).

**Because the gate cannot yet drive chatgpt's suite for you, adopt under
test-before-bump:** add the submodule, install the shims, run chatgpt's
`--offline` suites (`npm test` + top-level `*-test.js`, per `xrl4`), and only
commit the pin if green. The byte-identity makes a green outcome very likely, but
*your* suite is the authority for *your* pin.

## The shim (identical pattern per module)

```js
'use strict';
module.exports = require('<relative-path-from-table>');
```

## How to adopt (suggested)

```bash
cd chatgpt-webctl
git checkout -b adopt-base-v0.2.0
git submodule add git@github.com:CLIAI/base-webctl.git vendor/base-webctl
git -C vendor/base-webctl checkout v0.2.0   # == 5032279 (pushed + tagged on origin)
# replace each lib file above with its shim
npm test && for t in *-test.js; do node "$t"; done   # expect green
git add .gitmodules vendor/base-webctl lib/...
git commit
```
