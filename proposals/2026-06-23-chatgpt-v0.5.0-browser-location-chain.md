## What this does

Replace the **five** remaining `browser-location/` modules with re-export shims
onto base's factories. chatgpt's driver was the **canonical** body base ported
(strict superset of linkedin; it carries the WAY-1 upload staging). Your
`client-config.constants.js` and `file-staging.js` stay unchanged. Completes the
`browser-location/` migration (mounts shipped in v0.4.0).

| Module | base factory | shim injects |
|---|---|---|
| `browser-location/mounts.js` | `mounts.createMounts(C, o)` | constants + `dockerfilesDir` |
| `browser-location/chromium-docker-xpra.js` | `chromiumDockerXpra.createChromiumDockerXpra(C, o)` | constants + **`fileStaging`** + `dockerfilesDir` + `version` |
| `browser-location/registry.js` | `registry.createRegistry(C, o)` | constants + **`fileStaging`** + `dockerfilesDir` + `version` |
| `browser-location/xpra-attach.js` | `xpraAttach.createXpraAttach(C)` | constants |
| `browser-location/index.js` | `browserLocation.createBrowserLocation(C, o)` | constants + **`fileStaging`** + `dockerfilesDir` + `version` |

**chatgpt injects `fileStaging: require('./file-staging')`** into the driver (and
the registry/index that build it). That enables the WAY-1 upload path exactly as
today: per-slug host staging dir copy-staged + bind-mounted read-only at
`/cliai-uploads`, the `UPLOAD_STAGE_ENV` container var set, and `inspect()`
exposing `uploadHostDir`/`uploadContainerDir`. Your `file-staging.js` stays
consumer-owned (host-side staging/cleanup); base never requires it.

## xpra-attach â€” small behaviour improvement (linkedin was canonical here)

base shipped **linkedin's** xpra-attach body: the "xpra not installed" hint
interpolates the parameterized `opts.html5Port` (default `port+1`) instead of your
hardcoded `14501`. At default ports this is identical (14500â†’14501); at custom
ports the hint now shows the correct HTML5 port. No regression.

## The `version` + `dockerfilesDir` seams â€” read this

Two values were **repo-relative** and break once base is vendored:
`dockerfilesDir()` (`../../dockerfiles`, used by your `docker-cmd.js` +
`chromium-docker-xpra` at build time) and the lock's
`require('../../package.json').version`. base now takes both as injected opts; the
shims supply them from **your** repo. Don't rely on the fallbacks. Optional polish
(not required): have `file-staging.js` import base's `mounts.CONTAINER_UPLOAD_DIR`
to de-duplicate the `/cliai-uploads` literal.
